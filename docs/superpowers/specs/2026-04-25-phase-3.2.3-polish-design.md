# Phase 3.2.3 — Polish Design Spec

**Date:** 2026-04-25
**Status:** Approved (brainstorming complete, ready for implementation plan)
**Phase:** 3.2.3 of the 3.2.x sub-phases (Foundation → Interactivity → **Polish**)
**Prerequisite:** Phase 3.2.2 complete (HEAD `74f1785` or later — interactive lahan lifecycle shipped, 123 backend tests + 19 frontend tests baseline).

---

## 1. Goals & Non-Goals

### Goals
- **DnD kanban** (cross-column only): drag a task card from one column to another → card lands at the end of the target column. Reuses existing `useUpdateTask` mutation (already optimistic for `column_id`-only updates).
- **Multi-attempt tabs**: replace the current `expandedAttemptId` row-toggle pattern with a shadcn `<Tabs>` UI. Each attempt is a tab; switching tabs swaps the SSE log viewer.
- **Inline edit lahan title** via a hover-pencil affordance in the drawer header. Title input removed from `LahanInfoSection` form (canonical edit = pencil).
- **Tags chip input + linked_files textarea** in the drawer info section. Drawer becomes fully edit-capable (no need for curl/PATCH from the command line for any field).
- **`rust-embed` wiring** behind a Cargo feature flag `embed-ui`. Default `cargo build` unchanged. `cargo build --release --features embed-ui` produces a single binary that serves both `/api/*` and the embedded React UI with SPA fallback.
- **Demo-able end state:** `npm run build && cargo build --release --features embed-ui && kulisawit serve --port 7700` → open `http://localhost:7700` (no separate `vite dev`) → drag, rename, edit tags, dispatch, watch live logs across tabs.

### Non-Goals (3.2.3 specifically)
- **No within-column DnD reorder.** Cross-column moves only. (Within-column reorder requires backend position rebalance — deferred.)
- **No backend `GET /api/projects/:id/files`.** Linked files = plain text input, one path per line. Autocomplete deferred.
- **No accessibility audit pass.** (Pre-v0.1 polish; cross-cutting.)
- **No light theme / theme toggle.** Still dark-only per session style lock.
- **No `cargo-dist` setup.** Phase 5 owns install scripts + GitHub Releases.
- **No real kuli adapters.** Phase 4. Agent picker stays hardcoded `["mock"]`.
- **No project create UI.** Still curl-only; defer to a small follow-up task.
- **No DnD unit tests.** dnd-kit + jsdom is too painful for the value. Manual smoke covers it; revisit when Playwright lands (Phase 5+).
- **No lahan delete.** Intentional YAGNI carried over from 3.2.2.

## 2. Top-Level Decisions

| Decision | Choice | Rationale |
|---|---|---|
| Sub-phase scope | All 5 polish items in one sub-phase (DnD, tabs, inline edit, tags+files editor, rust-embed) | User-approved. Closes the 3.2 trilogy with a demo-able single binary. |
| DnD scope | Cross-column only | Backend untouched (existing `move_to_column` assigns next position). Within-column reorder is a non-trivial backend change — defer until clearly needed. |
| DnD library | `@dnd-kit/core` + `@dnd-kit/utilities` | Standard React DnD; works with shadcn idioms; battle-tested. |
| DnD activation | `PointerSensor` with `activationConstraint: { distance: 8 }` | Critical so click-to-open-drawer still works on TaskCard (without a distance constraint, every click registers as drag start). |
| Multi-attempt UX | shadcn `<Tabs>` (one tab per attempt; one log viewer at a time) | User-approved. Familiar, scales to 5-10 attempts via horizontal-scroll TabsList. Replaces 3.2.2's expand/collapse row pattern. |
| Inline edit pattern | Hover-pencil on drawer header title | Discoverable affordance; doesn't conflict with TaskCard click-to-open-drawer; keeps the kanban surface read-only-clickable. |
| `LahanInfoSection` cleanup | Remove title input from form (delegated to `EditableTitle`); keep description, column, tags, files | One canonical place to edit title — the pencil. Avoids dual edit paths fighting each other. |
| Tags editor | Chip input — Enter or comma adds, × removes | Standard pattern. Trim + dedup. |
| Linked files editor | Textarea, one path per line; blur parses to `string[]` | Simplest workable input. No backend coupling. Typo-prone but acceptable for v0.1. |
| `rust-embed` strategy | Cargo feature flag `embed-ui` (default off) | Dev workflow unchanged (vite dev + `cargo build`). Release path: `npm run build && cargo build --release --features embed-ui`. |
| SPA fallback | `embed-ui` handler serves `index.html` for any non-`/api/` path that doesn't match a baked asset | React Router-friendly even though we don't use router yet. |
| DnD testing | Manual smoke only | jsdom + dnd-kit pointer interactions = poor ROI. Documented out-of-scope; Playwright revisit in Phase 5+. |

```rust
//! Embedded UI assets — only compiled when `embed-ui` feature is on.

use axum::body::Body;
use axum::extract::Path;
use axum::http::{header, HeaderValue, StatusCode};
use axum::response::IntoResponse;
use axum::routing::get;
use axum::Router;
use rust_embed::Embed;

use crate::AppState;

#[derive(Embed)]
#[folder = "../../ui/dist/"]
struct Assets;

pub fn routes() -> Router<AppState> {
    Router::new()
        .route("/", get(serve_index))
        .route("/*path", get(serve_asset))
}

async fn serve_index() -> impl IntoResponse {
    serve_file("index.html")
}

async fn serve_asset(Path(path): Path<String>) -> impl IntoResponse {
    // Don't intercept /api/* — those routes are merged separately.
    if path.starts_with("api/") {
        return (StatusCode::NOT_FOUND, "Not Found").into_response();
    }
    if Assets::get(&path).is_some() {
        serve_file(&path)
    } else {
        // SPA fallback: any unknown frontend route → index.html
        serve_file("index.html")
    }
}

fn serve_file(path: &str) -> axum::response::Response {
    match Assets::get(path) {
        Some(content) => {
            let mime = mime_guess::from_path(path).first_or_octet_stream();
            let mut response = (StatusCode::OK, Body::from(content.data.into_owned()))
                .into_response();
            response.headers_mut().insert(
                header::CONTENT_TYPE,
                HeaderValue::from_str(mime.as_ref()).unwrap_or_else(|_| {
                    HeaderValue::from_static("application/octet-stream")
                }),
            );
            response
        }
        None => (StatusCode::NOT_FOUND, "Not Found").into_response(),
    }
}
```

```rust
pub async fn update_metadata(
    pool: &DbPool,
    id: &TaskId,
    tags: &[String],
    linked_files: &[String],
) -> DbResult<()> {
    let now = Utc::now().timestamp_millis();
    let id_str = id.as_str();
    let tags_json = serde_json::to_string(tags)?;
    let files_json = serde_json::to_string(linked_files)?;
    sqlx::query!(
        "UPDATE task SET tags = ?, linked_files = ?, updated_at = ? WHERE id = ?",
        tags_json,
        files_json,
        now,
        id_str
    )
    .execute(pool)
    .await?;
    Ok(())
}
```

Plan-writing fact-finding confirms whether this exists; if so, skip the db file changes.

### 5.1 `KanbanBoard` — DnD wrapper

```typescript
import {
  DndContext,
  DragOverlay,
  PointerSensor,
  useSensor,
  useSensors,
  type DragEndEvent,
  type DragStartEvent,
} from "@dnd-kit/core";

// inside the component
const sensors = useSensors(
  useSensor(PointerSensor, {
    activationConstraint: { distance: 8 },
  })
);
const [draggingTask, setDraggingTask] = useState<Task | null>(null);
const update = useUpdateTask();

function handleDragStart(evt: DragStartEvent) {
  const taskId = evt.active.id as string;
  const task = data?.columns.flatMap((c) => c.tasks).find((t) => t.id === taskId) ?? null;
  setDraggingTask(task);
}

function handleDragEnd(evt: DragEndEvent) {
  setDraggingTask(null);
  const taskId = evt.active.id as string;
  const overId = evt.over?.id as string | undefined;
  if (!overId || !overId.startsWith("column:")) return;
  const targetColumnId = overId.slice("column:".length);
  const sourceTask = data?.columns.flatMap((c) => c.tasks).find((t) => t.id === taskId);
  if (!sourceTask || sourceTask.column_id === targetColumnId) return;
  update.mutate({ id: taskId, body: { column_id: targetColumnId } });
}

return (
  <DndContext sensors={sensors} onDragStart={handleDragStart} onDragEnd={handleDragEnd}>
    <div className="grid …">
      {data.columns.map((col) => (
        <KanbanColumn key={col.id} column={col} projectId={projectId} />
      ))}
    </div>
    <DragOverlay>
      {draggingTask ? <TaskCard task={draggingTask} dragging /> : null}
    </DragOverlay>
  </DndContext>
);
```

### 5.4 `EditableTitle` — hover-pencil

```typescript
import { useState } from "react";
import { Pencil } from "lucide-react";
import { Button } from "@/components/ui/button";
import { Input } from "@/components/ui/input";

interface Props {
  value: string;
  onSave: (newValue: string) => void;
  pending?: boolean;  // disable during save
}

export function EditableTitle({ value, onSave, pending }: Props) {
  const [editing, setEditing] = useState(false);
  const [draft, setDraft] = useState(value);

  if (!editing) {
    return (
      <div className="group flex items-center gap-2">
        <span className="text-lg font-semibold">{value}</span>
        <Button
          type="button"
          variant="ghost"
          size="icon"
          aria-label="Edit title"
          className="opacity-0 group-hover:opacity-100 group-focus-within:opacity-100 h-6 w-6"
          onClick={() => {
            setDraft(value);
            setEditing(true);
          }}
        >
          <Pencil className="h-3 w-3" />
        </Button>
      </div>
    );
  }

  const cancel = () => {
    setDraft(value);
    setEditing(false);
  };

  const save = () => {
    if (draft.trim() && draft !== value) onSave(draft.trim());
    setEditing(false);
  };

  return (
    <div className="flex items-center gap-2">
      <Input
        autoFocus
        value={draft}
        onChange={(e) => setDraft(e.target.value)}
        onKeyDown={(e) => {
          if (e.key === "Enter") save();
          if (e.key === "Escape") cancel();
        }}
        disabled={pending}
        className="h-8"
      />
      <Button size="sm" onClick={save} disabled={pending || !draft.trim()}>
        Save
      </Button>
      <Button size="sm" variant="ghost" onClick={cancel} disabled={pending}>
        Cancel
      </Button>
    </div>
  );
}
```

### 5.5 `TagsInput` — chip input

```typescript
import { useState } from "react";
import { X } from "lucide-react";
import { Badge } from "@/components/ui/badge";
import { Input } from "@/components/ui/input";

interface Props {
  value: string[];
  onChange: (tags: string[]) => void;
}

export function TagsInput({ value, onChange }: Props) {
  const [draft, setDraft] = useState("");

  const addTag = (raw: string) => {
    const trimmed = raw.trim();
    if (!trimmed || value.includes(trimmed)) return;
    onChange([...value, trimmed]);
    setDraft("");
  };

  const removeTag = (tag: string) => {
    onChange(value.filter((t) => t !== tag));
  };

  return (
    <div className="flex flex-col gap-2">
      <div className="flex flex-wrap gap-1">
        {value.map((tag) => (
          <Badge key={tag} variant="secondary" className="text-xs flex items-center gap-1">
            {tag}
            <button
              type="button"
              aria-label={`Remove ${tag}`}
              onClick={() => removeTag(tag)}
              className="hover:text-destructive"
            >
              <X className="h-3 w-3" />
            </button>
          </Badge>
        ))}
      </div>
      <Input
        value={draft}
        onChange={(e) => setDraft(e.target.value)}
        onKeyDown={(e) => {
          if (e.key === "Enter" || e.key === ",") {
            e.preventDefault();
            addTag(draft);
          }
        }}
        placeholder="Add a tag and press Enter"
      />
    </div>
  );
}
```

### 5.6 `LinkedFilesInput` — textarea

```typescript
import { useState, useEffect } from "react";
import { Textarea } from "@/components/ui/textarea";

interface Props {
  value: string[];
  onChange: (files: string[]) => void;
}

export function LinkedFilesInput({ value, onChange }: Props) {
  const [text, setText] = useState(value.join("\n"));

  // If parent value changes (e.g. on drawer reopen), reset local text.
  useEffect(() => {
    setText(value.join("\n"));
  }, [value]);

  const commit = () => {
    const parsed = text
      .split("\n")
      .map((s) => s.trim())
      .filter(Boolean);
    // Avoid no-op churn
    if (JSON.stringify(parsed) !== JSON.stringify(value)) {
      onChange(parsed);
    }
  };

  return (
    <Textarea
      value={text}
      onChange={(e) => setText(e.target.value)}
      onBlur={commit}
      placeholder="src/parser.rs&#10;src/lexer.rs"
      rows={3}
    />
  );
}
```

### 5.7 `LahanInfoSection` — refactored (no title)

```typescript
import { useState, useEffect } from "react";
import { Button } from "@/components/ui/button";
import { Textarea } from "@/components/ui/textarea";
import { Label } from "@/components/ui/label";
import {
  Select, SelectContent, SelectItem, SelectTrigger, SelectValue,
} from "@/components/ui/select";
import { TagsInput } from "@/components/TagsInput";
import { LinkedFilesInput } from "@/components/LinkedFilesInput";
import { useUpdateTask } from "@/hooks/useUpdateTask";
import type { BoardColumn, Task, UpdateTaskRequest } from "@/types/api";

interface Props {
  task: Task;
  columns: BoardColumn[];
}

export function LahanInfoSection({ task, columns }: Props) {
  const [description, setDescription] = useState(task.description ?? "");
  const [tags, setTags] = useState<string[]>(task.tags);
  const [linkedFiles, setLinkedFiles] = useState<string[]>(task.linked_files);
  const update = useUpdateTask();

  // Sync local state when task prop changes (e.g. switching tasks in drawer)
  useEffect(() => {
    setDescription(task.description ?? "");
    setTags(task.tags);
    setLinkedFiles(task.linked_files);
  }, [task.id]);

  const descChanged = description !== (task.description ?? "");
  const tagsChanged = JSON.stringify(tags) !== JSON.stringify(task.tags);
  const filesChanged =
    JSON.stringify(linkedFiles) !== JSON.stringify(task.linked_files);
  const dirty = descChanged || tagsChanged || filesChanged;

  const save = () => {
    const body: UpdateTaskRequest = {};
    if (descChanged) body.description = description;
    // NOTE: tags and linked_files are NOT yet supported by PATCH /api/tasks/:id
    // (Phase 3.2.2 plan only wired title/description/column_id). Backend will
    // accept-and-ignore unknown fields, so save still updates description; tags
    // and files persist locally but won't round-trip until the backend handler
    // is extended in this same task. See §3.7 below.
    if (tagsChanged) (body as Record<string, unknown>).tags = tags;
    if (filesChanged) (body as Record<string, unknown>).linked_files = linkedFiles;
    if (Object.keys(body).length === 0) return;
    update.mutate({ id: task.id, body });
  };

  const onColumnChange = (columnId: string) => {
    update.mutate({ id: task.id, body: { column_id: columnId } });
  };

  return (
    <div className="flex flex-col gap-3">
      <div className="flex flex-col gap-1.5">
        <Label htmlFor="lahan-description">Description</Label>
        <Textarea
          id="lahan-description"
          rows={4}
          value={description}
          onChange={(e) => setDescription(e.target.value)}
        />
      </div>
      <div className="flex flex-col gap-1.5">
        <Label htmlFor="lahan-column">Column</Label>
        <Select value={task.column_id} onValueChange={onColumnChange}>
          <SelectTrigger id="lahan-column" aria-label="Column">
            <SelectValue />
          </SelectTrigger>
          <SelectContent>
            {columns.map((c) => (
              <SelectItem key={c.id} value={c.id}>{c.name}</SelectItem>
            ))}
          </SelectContent>
        </Select>
      </div>
      <div className="flex flex-col gap-1.5">
        <Label>Tags</Label>
        <TagsInput value={tags} onChange={setTags} />
      </div>
      <div className="flex flex-col gap-1.5">
        <Label>Linked files</Label>
        <LinkedFilesInput value={linkedFiles} onChange={setLinkedFiles} />
      </div>
      {update.isError && (
        <div role="alert" className="text-xs text-destructive">
          Failed to save changes
        </div>
      )}
      <div className="flex justify-end">
        <Button onClick={save} disabled={!dirty || update.isPending}>
          {update.isPending ? "Saving…" : "Save"}
        </Button>
      </div>
    </div>
  );
}
```

## 6. Data Flow

### 6.1 DnD

```
User drags TaskCard "t1" from column "c1" → drops on "column:c2"
  ↓
DndContext.onDragEnd({active.id="t1", over.id="column:c2"})
  ↓
strip prefix → targetColumnId = "c2"; read sourceColumnId from board cache
  ↓
sourceColumnId !== targetColumnId → useUpdateTask.mutate({id: "t1", body: {column_id: "c2"}})
  ↓ (optimistic — Phase 3.2.2 hook handles cache flip)
  Cache: card moves to c2 immediately
  ↓
PATCH /api/tasks/t1 { column_id: "c2" }
  ↓
On success: onSettled invalidates ["board"]
On error:   snapshot restored; cache reverts; ApiError surfaces (no inline UI in DnD path; revert is silent enough)
```

### 6.2 Inline title edit

```
Hover drawer header → pencil opacity 0 → 100
Click pencil → setEditing(true), draft=value, autofocus Input
Type "new title", Enter
  → onSave("new title")
  → useUpdateTask.mutate({id, body: {title: "new title"}})
  → text-only PATCH → pessimistic (button disabled, "Saving…")
  → On success: invalidate ["task", id] + ["board"]; SheetTitle re-renders
  → On error:   inline destructive alert (handled at LahanDetailSheet level; see §5.9)
```

### 6.3 Tabs lifecycle

```
User opens drawer → useTaskAttempts → data with 3 attempts
LahanAttemptsSection mounts → Tabs default value = last attempt
  → only that <TabsContent> renders → AttemptLogViewer mounts → EventSource opens
User clicks Tab 1 → expandAttempt("a1")
  → previous TabsContent unmounts → AttemptLogViewer unmounts → useEffect cleanup → es.close()
  → new TabsContent mounts → new EventSource opens for a1
```

The lifecycle works correctly because the existing `useAttemptEvents` hook (Phase 3.2.2) already cleans up on unmount.
